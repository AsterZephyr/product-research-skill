# Analysis Frameworks Reference

产品调研中使用的分析框架详细说明。SKILL.md 中引用此文件获取框架细节。

## 投资人视角框架

### TAM-SAM-SOM 市场规模估算

| 层级 | 定义 | 估算方法 |
|------|------|---------|
| **TAM** (Total Addressable Market) | 目标产品/服务的全球市场总量 | 自上而下：行业报告 + CAGR 推算 |
| **SAM** (Serviceable Addressable Market) | 产品实际可触达的市场 | 按地域、客户类型、价格段筛选 TAM |
| **SOM** (Serviceable Obtainable Market) | 短期内可获取的市场份额 | 基于现有客户、增长率、竞争格局估算 |

数据来源优先级：Gartner/IDC/Frost & Sullivan 报告 > 投行研报 > 公司自述 > 推算

### Business Model Canvas（商业模式画布）

9 要素简版输出：

| 要素 | 需回答的问题 |
|------|-------------|
| 价值主张 | 解决什么问题？为什么选你而不是替代品？ |
| 客户群体 | 谁在用？谁在付费？（可能不同） |
| 渠道 | 用户怎么发现和接触产品？ |
| 客户关系 | 自助 / 辅助 / 社区 / 专属服务？ |
| 收入模式 | SaaS / 交易抽成 / 广告 / 增值服务？ |
| 关键资源 | 技术壁垒、数据资产、品牌、团队 |
| 关键活动 | 持续投入的核心工作 |
| 关键合作 | 上游供应商、分发渠道、战略合作 |
| 成本结构 | 固定成本 vs 变动成本，规模效应如何 |

### Porter's Five Forces（波特五力）

| 力量 | 分析要点 |
|------|---------|
| 现有竞争 | 竞争者数量、差异化程度、行业增速 |
| 新进入者威胁 | 进入门槛（资金/技术/品牌/网络效应） |
| 替代品威胁 | 用户的替代方案是什么？切换成本多高？ |
| 供应商议价力 | 关键供应商是谁？是否可替代？ |
| 买家议价力 | 客户集中度、价格敏感度、切换成本 |

评分标准：低 / 中 / 中高 / 高，附具体说明。

### Bull Case vs Bear Case

- **Bull Case**（看多理由）：差异化、壁垒、增长飞轮、团队、市场时机
- **Bear Case**（看空理由）：竞争、依赖、变现、监管、执行风险

要求双方论点均衡，不偏不倚。

## 产品视角框架

### User Persona（用户画像）

每个画像包含：

| 字段 | 内容 |
|------|------|
| 画像名称 | 一个典型角色 |
| 描述 | 谁，在哪工作，日常工作内容 |
| 核心痛点 | 现有方案解决不好的问题 |
| 产品解法 | 本产品如何解决 |

建议覆盖 3-4 个画像（主力用户 + 次要用户 + 潜在用户）。

### AARRR 漏斗

| 阶段 | 关键问题 | 指标 |
|------|---------|------|
| **Acquisition** | 用户从哪来？ | 渠道、CAC |
| **Activation** | 首次体验如何？ | 注册转化率、首次使用完成率 |
| **Retention** | 用户会回来吗？ | DAU/MAU、留存曲线 |
| **Revenue** | 怎么赚钱？ | ARPU、LTV、付费转化率 |
| **Referral** | 用户会推荐吗？ | NPS、病毒系数 |

### Jobs-to-be-Done（JTBD）

格式：

```
当 [情境] 时，我想 [动作]，以便 [期望结果]。
```

按优先级排序（P0/P1/P2），评估产品完成度（高/中/低）。
